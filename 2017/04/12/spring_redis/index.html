Redis在Java web中的应用

发表于 2017-04-12   |   分类于 Java Web   |     |   阅读次数

前情概要，项目中遇到需要使用redis的情况

之前的项目中，使用了微服务架构，因此session的管理变得略微繁琐，因为每次请求中，session可能不尽相同，因此需要识别session，则需要在请求头中添加一个用户身份的标记(也就是我们所说的Token）,服务器通过拦截器来验证请求头中是否存在这个Token，以此达到权限拦截的目的。

那么服务器中保存的token到底要怎么存储呢，当长时间的未使用时，我们需要把它从存储的地方删除，因此之前我们采取过一下几种办法：

  1. 全局的一个map
  2. ThreadLocal
  3. Redis

对这是历经2次重构后，才使用的redis,很明显，第一种线程不安全，且容易内存泄漏=-=，考虑了一下然后使用了第二种，不得不说ThreadLocal对于一个Web项目来说，是一个挺好的东西，它完美解决了我们所需要的获取这个请求中用户信息的需求，但是，我们考虑到项目重启之后之前存在的记录不都没有了么！但是我们又想保留这些记录要怎么办？那么redis的想法由此诞生。

Redis简介

redis是一个key-value存储系统。和Memcached类似，它支持存储的value类型相对更多，包括string(字符串)、list(链表)、set(集合)、zset(sorted set –有序集合)和hash（哈希类型）。这些数据类型都支持push/pop、add/remove及取交集并集和差集及更丰富的操作，而且这些操作都是原子性的。在此基础上，redis支持各种不同方式的排序。与memcached一样，为了保证效率，数据都是缓存在内存中。区别的是redis会周期性的把更新的数据写入磁盘或者把修改操作写入追加的记录文件，并且在此基础上实现了master-slave(主从)同步。Redis是一个高性能的key-value数据库。

以上摘自百度百科，其实简单来说，redis是一种map结构的高速缓存。
